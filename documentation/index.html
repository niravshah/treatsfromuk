Special Builder documentation

We have a made a solid effort to provide as much documentation and information about Special HTML Builder, the way it was built, the way it works and its features. If, however, you find anything lacking from this document or you have any questions, do not hesitate to contact us at help.designsvilla@gmail.com or submit a ticket through our help desk located at DesignsVilla Support.

Introduction

What it is

Special HTML Builder is a Javascript application which allows basic content editing visually.

HTML Builder comes bundled with several HTML blocks to demonstrate it's usage (these are the same blocks as used in the online demo). You are free to use the provided blocks in whatever way you like.

HTML Builder runs (almost) entirely in the browser. The only server side code comes in the form of four PHP files, used for saving a page, uploading images, exporting an entire site and previewing changes.

What it's not

HTML Builder is NOT a CMS (content management system). It does not provide a way for end-users to work on live websites and make changes to such websites.

Requirements

To be able to use HTML Builder, you must have the following:

  • PHP
  • Apache webserver (Windows servers will probably work, but are not supported)
  • An FTP tool to upload the files

Installation

Installing the HTML Builder script is simply a matter uploading the contents of the builder folder to a server/hosting account. Once uploaded, point your browser to said server/hosting and your script will work.


For example: www.yourdomain.com/builder or if you are using on subdomain url would be something like this : www.sub.yourdomain.com/builder

*Note: please don't upload the .zip file you downloaded from envato directly to your server, first unzip it, then upload the folder "builder" to the server.

Update

Please follow these steps below To update the builder to the latest version:

  1. Download upload folder from "/elements/images/uploads/".
  2. AFTER downloading upload folder Delete builder folder from server.
  3. Upload the new builder to your server.
  4. now again upload previously downloaded upload folder to "/elements/images/".

Drag & Drop

HTML Builder allows users to drag HTML blocks onto a canvas and drop them where they'd like them to appear. Once dropped onto the canvas, the position can be changed by dragging and dropping as well. Blocks can be delete individually or all blocks on the current page can be deleted in one go.

Editing content

The HTML Builder script can also be used to edit written content inside each element. HTML Builder uses a rich text editor, allowing to make changes to the written content. Depending on what type of element you're editing, certain options will be disabled in the editor. certain text elements might not be editable.

Editing styles

The HTML Builder script also allows for basic CSS editing. Using the built-in CSS editor, end-users can customize whatever CSS attributes the script's admin has configured. When editing a link, the end-user can point the link to either an internal page or an (external) URL.

Which CSS attributes are active is configured through the builder.js editableItems object.

Editing links

The HTML Builder script allows user to edit links as well. To be able to edit links, firstly the link selectors will need to be added to the "editableItems" array in the /js/builder.js file. To edit links on the canvas, firstly select the Details building mode, this will allow you to select links. When selected, a "link" tab will be visible in the left panel, allowing you to either choose a page to link to or enter a URL manually.

Linking within a single page
When building a one-page website, you might want to link to sections within a single page rather then to a separate page. To do this, simply edit the link and use the second dropdown to choose the block you'd like to link to. The ID's to which can be linked, have to be configured within the block HTML files, and the ID attribute should be assigned to the first child element of the main container (for an example, please have a look at any of the bundled block files).


Editing images

The HTML Builder script allows user to edit images as well. To edit images on the canvas, Select an image from canvas When selected, a "image" tab will be visible in the left panel, allowing you to either upload an image or enter a URL manually.
- To change the image, input new image url inside url(), for example:
url(yourdomain.com/image.jpg)
- To remove the image, just input "url()"


Parallax Background

* To make the background image parallax, set the "background-size to cover" and "background-attachment to fixed".

Cloning elements

Items which have been setup in the "editableItems" object in /js/builder.js can be cloned. select the item which you want to clone and finally click the clone button from the left panel. Please note that cloned items will be inserted into the canvas right next to the original item.

Exporting a site

Whenever the user is happy with the final result, the created markup can be exported by click the green export button. The export feature will grab all the used HTML elements for each created page. Once all pages are completed, all the HTML files are bundled into a ZIP archive together with the other external resources like stylesheets, Javascript files and images which will be downloaded by the end user.


Multiple pages

The HTML Builder script allows users to create multiple pages at once. To add a new page, simply click the green button labeled "+ Add" just below the current pages list. You can alter the page names as well (please use names without the .html extension, these are added automatically during the export). When exporting the markup, the pages will use the same names. To delete a page, simply click the little delete icon next to the page you'd like to delete.

Responsive views

HTML Builder allows users to switch between different view modes, this to ensure the resulting page will look good on mobiles, tablets and desktop screens. To toggle between modes, use the three buttons to the left of the main toolbar.

The screen sizes for the view modes can be configured in the builder.js module, by using the "responsiveModes" object.

Global blocks

HTML Builder allows users to configure so-called "global blocks". These are blocks that maintain the same appearance and settings across different pages. This feature comes in handy when configuring navigations and footers for example.

Each block on the canvas has a check box labeled "Global". When checked, changes made to this block will be applied to the same block on all pages in your site, assuming those other blocks are marked as "global" as well.

Field setting

You can setup form fields via builder by following steps below.
  • darg and drop desired form section in canvas.
  • click on any filed, left panel will be open, in left panel click on field setting tab.
  • in field setting tab you will find 3 options, (field name, field placholder, and select box for whether this field is required or not
  • fill the all options as per your requirment and hit appy button
  • *Note:you can select diffirent service provider for each page if you are using multiple forms on same page.
  • * Note: this area is important to make mailchimp and other services work, user field name attribute as per input attribute in your mailchimp and others services forms.




Form setting
You can setup form via builder by following steps below.
  • darg and drop desired form section in canvas.
  • select form wrapper by clicking on form box not on field, left panel will be open, in left panel click on form setting tab.
  • in form setting tab choose any service provider from select box
  • hit apply changes button, your form is ready
  • * Note: You must also select service provider and fill form data on export tab




Form Export setting
You can setup form export data via builder by following steps below.
  • After completing your work click on export button
  • On export tab after filling seo settings, select form type from select box
  • Then select desired service provider from left menu on export tab
  • after selecting service provider fill in required data
  • * Note: you can use multiple service provider on one page or multiple page, so fill in all required services info. and hit export button

After export form setting

If you are using one of the predefined landing pages or you exported the landing page from the builder and you want to edit the email service provider for the forms in the page simply follow these steps:

  1. Browse to the mail folder".
  2. Open integration.php in any text editor".
  3. In line number 2 input the email service provider that you want to use as following:
    • "php" for Custom Email.
    • "dvmailchimp" for Mailchimp.
    • "dvCampaginMonitor" for Campaign Monitor.
    • "dvGetResponse" for GetResponse.
    • "dvAWeber" for AWeber.
    • "dvActiveCampaign" for Active Campaign.
    • "dvMailerLite" for MailerLite.
  4. For the used email service provider input the required information as following:
    • replace your Email address on line number 4.
    • Email subject on line number 5
    • after that input api key and list ids for each rewuired service from line number 13 to onward
  5. Save integration.php to make your form ready to use
